Project Management

Having the most experience on the Unity engine led me to work as a facilitator for my teamates. My great interest in gameThus making me led some of the asset production

Development

During the project, I worked on the following features:

  • AI agent navigation (Aerial and Terrestrial) and interactions through a node system
  • Animation integration and animation driven movement using Unity blend trees
  • Slope Management
  • VR core gameplay mechanics implementation (Grab, Throw, Lever, Button, PlacePoint)
  • Game Optimisation for Quest 2 using Unity profiling tools, Mesh Baker and Amplify Impostors

What I learned from this experience

My 6 months internship in Dreamirl showed me the insides of an Indie game studio, thus allowing me to test my skills in a professional game making environment. The following

  • Problems need to be raised as soon as they emerge. Some problematics could have been resolved more efficiently by sharing them with the team in the earlier stages of the project
  • Technology watch needs to be done before everything else. Several features were developed from scratch while plugins could have saved us a lot of time
  • Build as much as you can. The goal was a release on Quest 2, but builds were made too late in the planning. Some features didn’t work as expected on the device, and optimization problematics emerged too close to the release date
  • Communication is key. Giving feedback on each others work highly improved group cohesion and the overall project quality
